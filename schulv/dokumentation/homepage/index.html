Einleitung

Am 30. Januar 2002 wurde für mit Hilfe von Berlios eine komplette Projektumgebung eingerichtet. Die komplette Entwicklung einer freien Software zur Schulverwaltung wird über diese Entwicklungsumgebung abgewickelt werden. Dazu werden mehrere Mittel angeboten, von denen wir erst mal folgende nutzen möchten:

  • Mailinglisten
  • CVS
  • Homepage
  • Neuankündigungen
  • Umfragen (siehe auch weiter unten)
Wer sich aktiv an Codeentwicklung beteiligen möchte, braucht natürlich Schreibzugriff auf CVS. Dies wäre auch nötig, falls jemand die Verwaltung der Homepage übernehmen will.
Damit einem Nutzer Schreibzugriff gewährt werden kann, muss er aber einen Berlios Nutzer Account haben, den man hier sehr einfach und schnell erstellen kann.

Momentan ist schon eine aktuelle Umfrage verfügbar, aber noch nicht in der Berlios Projektumgebung, sondern auf den Seiten des PingoS Projekt.
Diese Umfrage versucht die momentane Nutzung von Software zur Schulverwaltung zu erkunden und vorhandene Probleme zu sammeln. Diese Informationen sind eine Grundvorraussetzung für das SchulV Projekt.

Projektziel

 Für den Schulverwaltungsmarkt ist Linux aufgrund von fehlender Software noch keine Alternative. Das Ziel dieses Projektes ist es, vorhandene Lücken zu identifizieren und zu beseitigen, um den Schulverwaltungsmarkt für Linux zu erschliessen.

Um unser Ziel zu erreichen werden wir einerseits freie Software (im Sinne der LGPL/GPL) für die Schulverwaltung entwickeln und andererseits Migrationskonzepte erarbeiten. Desweiteren werden auch Angebote für Fortbildungen, sowie diverse Anforderungsprofile für Installations- und Wartungsangebote konzipiert.

Den größten Wert legen wir auf die Entwicklung von umsetzbaren Konzepten und keineswegs auf kostenlose Konzepte. Daher werden interessante kommerzielle Angebote von Anfang an in die Planung einbezogen.

Werkzeuge zur Entwicklung

 Mal ganz abgesehen von den diversen Compilern und Servern, hier eine kleine Liste von Werkzeugen und deren Einsatz, die bei der Entwicklung für dieses genutzt werden:
Werkzeug Bemerkungen
Dia Tool zum Zeichnen von Vektorgrafiken hauptsächlich im Bereich Software-Diagramme (UML, ERM ...)
Es gibt mit Sicherheit wesentlich umfangreichere Werkzeuge, die dem Zeichner mehr Unterstützung und Funktionalität bieten, aber der Reiz an Dia liegt in der Einfachheit (auch hinsichtlich der Bedienung)
LDAP Browser\Editor (ver 2.8.2) Ein sehr umfangreiches Programm zur Verwaltung von LDAP Verzeichnissen. Man kann es natürlich auch für OpenLDAP verwenden.
Dabei sollte man folgende Einstelloptionen zum OpenLDAP Server beachten, wenn man die slapd.conf und entsprechenden Schematas aus dem CVS verwendet:

  • Host: localhost bzw. der Rechnername auf dem openldap läuft
  • Port: 389 (wenn ihr nicht openldap auf einem anderen Port laufen habt)
  • Version: 3
  • Base DN: o=schule.edeal.de
  • SSL: nein
  • Anonymous bind: nein
  • User DN: cn=Manager,o=schule.edeal.de
  • append base DN: nein
  • Password: secret
GQ 0.4.0 Ein einfaches Programm zur Verwaltung von LDAP Verzeichnissen. Es ist sicher noch nicht ausgereift, bietet aber auch interessante Sichten auf die Daten.
